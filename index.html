User Management

useraddCréer un nouveau compte utilisateur de bas niveau.System
Commande standard pour instancier un nouvel utilisateur. Contrairement à 'adduser' (= script interactif), 'useradd' ne fait que ce qu'on lui demande via les options.
-m Crée automatiquement le répertoire personnel
-d Définit un home personnalisé
-s Choisit le shell de connexion
-u Attribue un UID précis au lieu du prochain UID libre
-g Groupe primaire
-G Groupe(s) secondaire(s)
-c Commentaire
-r Compte Système
			
1. $ useradd -m alice
# L'option '-m' (make-home) est cruciale. Sans elle, le dossier /home/alice ne sera pas créé.
2. $ useradd -m -s /bin/bash bob
# '-s' définit le shell par défaut. Si omis, le système peut assigner /bin/sh (très limité).
3. $ useradd -m -G sudo,docker,www-data charlie
# '-G' ajoute l'utilisateur à une liste de groupes secondaires séparés par des virgules (sans espaces).
4. $ useradd -r -s /bin/false service_user
# '-r' crée un compte système (UID inférieur à 1000). '-s /bin/false' empêche ce compte de se connecter physiquement (idéal pour faire tourner un service web ou une base de données en sécurité).
5. $ useradd -m -d /mnt/data/diane diane
# '-d' permet de définir un chemin personnalisé pour le répertoire personnel de l'utilisateur.
Références : [useradd @ doc.ubuntu-fr][useradd man @ pwet.fr][useradd man @ linux.die][useradd @ Malekal][gestion utilisateurs/groupes @ Malekal][useradd @ man7]
userdelSupprimer définitivement un compte utilisateur.
Supprime les entrées de l'utilisateur dans /etc/passwd et /etc/shadow. Attention aux fichiers "orphelins" qui appartiendront à un UID fantôme si le dossier personnel n'est pas supprimé.
-f, --force Force la suppression même si le compte est encore utilisé
-r, --remove Supprime aussi le home et le spool mail
-Z, --selinux-user Supprime l'entrée SELinux associée si présente
			
1. $ userdel alice
# Supprime le compte système, mais le dossier /home/alice et son contenu restent intacts sur le disque.
2. $ userdel -r bob
# '-r' (remove) est l'option la plus courante : elle supprime le compte, le dossier /home/bob, et le spool de mails de bob. Action irréversible.
3. $ userdel -f charlie
# '-f' (force) force la suppression du compte même si charlie est actuellement connecté ou a des processus en cours d'exécution.
Références : [userdel @ linux.die][userdel @ Ubuntu][userdel & deluser @ Oleks][suppression utilisateur @ Malekal][userdel @ man7]
usermodModifier les paramètres d'un compte existant.
Permet d'altérer tout ce qui a été défini (ou oublié) lors du `useradd`. C'est l'outil de maintenance de compte par excellence.
-l login (change le nom de l'utilisateur)
-d home (définit un nouveau répertoire personnel)
-m move (déplace le contenu de l'ancien home vers le nouveau)
-g gid (change le groupe primaire)
-aG append groups (ajoute des groupes secondaires sans supprimer les actuels)
-s shell (change le shell par défaut)
-u uid (change l'identifiant numérique unique)
-L / -U lock / unlock (verrouille ou déverrouille le compte)
            
1. $ sudo usermod -l alicia alice
# Renomme l'utilisateur 'alice' en 'alicia'.
2. $ sudo usermod -s /bin/bash developpeur
# Définit bash comme shell de connexion par défaut pour l'utilisateur 'developpeur'.
3. $ usermod -aG docker alice
# ATTENTION : '-a' (append) DOIT être utilisé avec '-G'. Cela "ajoute" alice au groupe docker. Si vous oubliez le '-a', alice sera retirée de TOUS ses autres groupes pour n'être que dans docker !
4. $ usermod -d /home/alicia -m alicia
# '-d' redéfinit le chemin du home. '-m' (move) déplace physiquement les anciens fichiers vers ce nouveau dossier.
5. $ usermod -L bob
# '-L' (Lock) verrouille le compte en insérant un '!' devant le mot de passe chiffré dans /etc/shadow. Connexion impossible.
6. $ usermod -U bob
# '-U' (Unlock) déverrouille un compte précédemment bloqué.
Références : [usermod @ Ubuntu][usermod @ linux.die][usermod @ Oleks][usermod @ Malekal][gestion utilisateurs/groupes @ Malekal][usermod @ man7]
passwdGérer les mots de passe et leur expiration.
Gère l'authentification. Un utilisateur normal ne peut changer que son propre mot de passe. Root peut changer n'importe lequel sans connaître l'ancien.
-l lock (verrouille le compte)
-u unlock (déverrouille le compte)
-d delete (supprime le mot de passe)
-e expire (force le changement immédiat)
-S status (affiche l'état du compte)
-n min days (délai min. entre changements)
-x max days (durée de validité max.)
-w warn days (alerte avant expiration)
			
1. $ passwd
# Change le mot de passe de l'utilisateur actuellement connecté. Le système demandera l'ancien mot de passe en premier.
2. $ sudo passwd alice
# Force le changement du mot de passe d'alice. L'ancien mot de passe n'est pas demandé.
3. $ sudo passwd -e bob
# '-e' (expire) fait expirer immédiatement le mot de passe actuel. À sa prochaine tentative de connexion, bob sera forcé d'en choisir un nouveau.
4. $ sudo passwd -S charlie
# '-S' (Status) affiche l'état du mot de passe de charlie (P = actif, L = verrouillé, NP = pas de mot de passe).
Références : [passwd @ Malekal][passwd @ man7][passwd @ Ubuntu]
groupaddInstancier un nouveau groupe d'utilisateurs.
Crée une nouvelle entité logique dans /etc/group à laquelle on pourra assigner des utilisateurs pour faciliter la gestion des droits sur des dossiers.
-g GID (identifiant numérique unique)
-o non-unique (autorise un GID en doublon)
-p password (mot de passe chiffré)
-r system (crée un groupe système)
-f force (quitte avec succès si le groupe existe déjà)
-K key=value (surcharge les limites par défaut)
            
1. $ sudo groupadd developpeurs
# Crée un nouveau groupe nommé 'developpeurs' avec le prochain GID disponible.
2. $ sudo groupadd -g 1500 tech
# Crée le groupe 'tech' en lui assignant spécifiquement l'identifiant 1500.
3. $ sudo groupadd -r www-data
# Crée un groupe système (généralement avec un GID inférieur à 1000).
4. $ sudo groupadd -f marketing
# Crée le groupe marketing, mais ne renvoie pas d'erreur s'il existe déjà.
Références : [groupadd @ Malekal][/etc/group @ Malekal][groupadd @ man7]
groupdelSupprimer un groupe existant.
Supprime l'entrée du groupe. Notez que cela ne supprime ni les utilisateurs qui en faisaient partie, ni les fichiers appartenant à ce groupe.
-h help (affiche l'aide)
-R root (applique les changements dans le répertoire chroot)
-f force (supprime même s'il est le groupe primaire d'un utilisateur)
            
1. $ sudo groupdel dev-team
# Supprime définitivement le groupe 'dev-team' du système.
2. $ sudo groupdel -f mon-groupe
# Force la suppression du groupe, même si des fichiers ou des processus l'utilisent encore (à utiliser avec prudence).
Références : [groupdel @ Malekal][/etc/group @ Malekal][groupdel @ man7]
groupmodModifier les paramètres d'un groupe.
Permet de changer le nom ou l'ID d'un groupe existant sans avoir à le recréer et réassigner ses membres.
-g new GID (modifie l'ID numérique)
-n new name (renomme le groupe)
-o non-unique (autorise un GID doublon)
-p password (change le mot de passe)
            
1. $ sudo groupmod -n dev-team developpeurs
# Renomme le groupe 'developpeurs' en 'dev-team'. Les membres restent les mêmes.
2. $ sudo groupmod -g 2000 dev-team
# Change le GID du groupe à 2000. Attention : les fichiers appartenant à l'ancien GID devront être mis à jour manuellement.
3. $ sudo groupmod -p '$6$rounds=5000...' staff
# Modifie le mot de passe chiffré du groupe staff.
Références : [groupmod @ Malekal][/etc/group @ Malekal][groupmod @ man7]
idAfficher l'identité et les groupes actifs.
Commande essentielle pour vérifier ses propres privilèges ou ceux d'un autre utilisateur après une modification avec usermod.
-u user (affiche uniquement l'UID numérique)
-g group (affiche uniquement le GID primaire)
-G groups (affiche tous les GID du compte)
-n name (affiche le nom au lieu du numéro - s'utilise avec -u, -g ou -G)
-a all (ignore pour la compatibilité avec d'autres versions)
-z zero (délimite les sorties par un caractère nul au lieu d'espaces)
            
1. $ id
# Affiche l'UID, le GID primaire et tous les groupes secondaires de l'utilisateur actuel.
2. $ id -un alice
# Affiche uniquement le nom d'utilisateur (utile dans les scripts).
3. $ id -G bob
# Liste tous les identifiants de groupes auxquels appartient bob.
4. $ id -nG
# Affiche les noms de tous les groupes de l'utilisateur actuel séparés par des espaces.
5. $ id -u
# N'affiche que le numéro d'UID. Souvent utilisé dans des scripts (ex: if [ $(id -u) -eq 0 ]; then echo "Je suis root"; fi).
Références : [id @ Malekal][id @ LinuxTricks][id @ man7]
suSubstitute User (Changer d'utilisateur en cours de session).
Ouvre un nouveau shell en tant qu'un autre utilisateur. Par défaut, cible l'utilisateur root.
- login (lance un shell de connexion avec l'environnement de l'utilisateur)
-l login (équivalent au tiret seul '-')
-c command (exécute une commande spécifique puis quitte)
-s shell (utilise un shell spécifique au lieu de celui par défaut)
-p preserve-environment (conserve les variables d'environnement actuelles)
            
1. $ su -
# Devient root et charge tout son environnement (PATH, variables, répertoire /root).
2. $ su alice
# Bascule vers le compte d'alice, mais garde l'environnement de l'utilisateur précédent.
3. $ su -l bob -s /bin/zsh
# Se connecte en tant que bob en utilisant spécifiquement le shell zsh.
4. $ su -c "systemctl restart nginx" root
# '-c' (command) permet d'exécuter UNE seule commande en tant que root, puis la session su se ferme automatiquement.
Références : [su @ Malekal][su vs sudo @ LinuxTricks][su @ man7]
sudoSuperuser DO (Exécuter avec élévation de privilèges).
Permet à un utilisateur autorisé d'exécuter une commande en tant que root sans connaître le mot de passe root (il utilise son propre mot de passe). Tout est logué dans le système.
-i login (simule une connexion initiale, charge l'environnement root)
-s shell (lance le shell spécifié par la variable SHELL)
-u user (exécute la commande en tant qu'un utilisateur spécifique)
-l list (liste les privilèges autorisés pour l'utilisateur)
-k kill (invalide les identifiants en cache, demande le mot de passe au prochain sudo)
-v validate (met à jour le cache du mot de passe pour 15 min par défaut)
            
1. $ sudo apt update
# Exécute la mise à jour des dépôts avec les privilèges root.
2. $ sudo -i
# Ouvre une session interactive en tant que root avec son environnement complet.
3. $ sudo -u alice ls /home/alice/docs
# Liste les fichiers d'un répertoire appartenant à alice en utilisant son identité.
4. $ sudo -l
# (list) Affiche quelles commandes vous avez le droit d'exécuter avec sudo (défini dans le fichier /etc/sudoers).
5. $ sudo !!
# Raccourci magique de bash : rappelle la dernière commande tapée et y préfixe 'sudo'. Très utile quand on a oublié de le mettre et qu'on reçoit une erreur "Permission denied".
Références : [sudo @ Malekal][sudo/sudoers @ LinuxTricks][sudo @ man7]

File & Directory Management - Navigation & Base

lsLister le contenu d'un répertoire.
'ls' (List) interroge le système de fichiers pour afficher les objets présents dans un dossier. C'est la commande la plus utilisée pour se repérer.
-l long (format détaillé)
-a all (affiche tout, y compris les fichiers cachés débutant par '.')
-A almost all (-a sans '.' et '..')
-d, --directorylist directory entries
-h human-readable (affiche les tailles en Ko, Mo, Go avec -l)
-t time (trie par date de modification, du plus récent au plus ancien)
-r reverse (inverse l'ordre du tri)
-R recursive (liste aussi le contenu des sous-répertoires)
-S size (trie par taille de fichier)
-1 one (affiche un seul fichier par ligne)
            
1. $ ls -la
# Liste tous les fichiers (cachés inclus) avec les détails complets.
2. $ ls -lh /var/log
# Affiche les fichiers de logs avec des tailles lisibles (ex: 4.2M au lieu de 4404019).
3. $ ls -R Documents/
# Explore récursivement tout ce qui se trouve dans le dossier Documents.
4. $ ls -lt
# '-t' trie par date de modification décroissante. Très utile pour voir quels fichiers viennent d'être créés ou modifiés dans un dossier encombré.
5. $ ls -lS
# '-S' (majuscule) trie par taille de fichier. Idéal pour identifier rapidement les fichiers qui saturent ton espace disque.
6. $ ls -R
# '-R' (Récursif) descend dans tous les sous-dossiers. Attention : si tu le lances à la racine (/), l'affichage ne s'arrêtera plus !
7. $ ls -d */
# N'affiche que les dossiers du répertoire courant, en ignorant les fichiers simples.
Références : [ls @ Malekal][ls @ GNU][ls @ man7]
cdChange Directory (Changer de répertoire).
Permet de déplacer le "point focal" de ton terminal dans l'arborescence.
-P physical (suit les liens symboliques vers leur dossier réel)
-L logical (suit les liens symboliques, comportement par défaut)
.. parent (remonte d'un niveau dans l'arborescence)
~  home (raccourci vers le répertoire personnel de l'utilisateur)
-  previous (retourne au dernier répertoire visité)
/  root (va à la racine du système de fichiers)
            
1. $ cd /var/www/html
# Déplacement via un chemin absolu (partant de la racine).
2. $ cd ../Documents
# Remonte d'un cran puis entre dans 'Documents' (chemin relatif).
3. $ cd ~
# Retour immédiat dans votre dossier personnel (/home/utilisateur).
4. $ cd -
# Bascule entre le répertoire actuel et le précédent (très pratique pour les allers-retours).
Références : [cd @ Bash Manual][navigation shell @ Bash Manual]
pwdPrint Working Directory (Afficher le chemin actuel).
Quand on est perdu dans des arborescences complexes, pwd donne le chemin complet depuis la racine.
-L logical (affiche le chemin incluant les liens symboliques, par défaut)
-P physical (affiche le chemin réel, en résolvant les liens symboliques)
--help aide (affiche l'aide de la commande)
--version version (affiche les informations de version)
            
1. $ pwd
# Affiche par exemple : /home/alice/Documents/Projets
2. $ pwd -P
# Si vous êtes dans un dossier qui est un lien symbolique, affiche le véritable emplacement sur le disque.
3. $ echo "Je suis ici : $(pwd)"
# Utilisation courante dans les scripts pour mémoriser l'emplacement actuel.
Références : [pwd @ GNU][pwd @ man7]
mkdirMake Directory (Créer un dossier).
Crée un ou plusieurs dossiers. Par défaut, il échoue si le dossier parent n'existe pas.
-p parents (crée les dossiers parents si nécessaire, sans erreur s'ils existent)
-m mode (définit les permissions au moment de la création, ex: 755)
-v verbose (affiche un message pour chaque répertoire créé)
-Z context (définit le contexte de sécurité SELinux)
            
1. $ mkdir nouveau_dossier
# Crée un dossier nommé 'nouveau_dossier' dans le répertoire actuel.
2. $ mkdir -p projets/python/scripts
# Crée toute l'arborescence d'un coup, même si 'projets' ou 'python' n'existent pas.
3. $ mkdir -m 777 public_folder
# Crée un dossier avec tous les droits en lecture/écriture/exécution pour tout le monde.
4. $ mkdir {janvier,fevrier,mars}_logs
# Astuce shell : crée trois dossiers d'un coup (janvier_logs, fevrier_logs, mars_logs).
Références : [mkdir @ Malekal][mkdir @ GNU][mkdir @ man7]
rmRemove (Supprimer des fichiers ou répertoires)
Attention : Commande destructive sans corbeille. Les flags -r (récursif) et -f (force) sont les plus puissants. 
-r recursive (supprime un répertoire et tout son contenu)
-f force (ignore les fichiers inexistants, ne demande jamais de confirmation)
-i interactive (demande confirmation avant chaque suppression)
-v verbose (affiche le nom de chaque fichier avant de le supprimer)
-d dir (supprime des répertoires vides, comme rmdir)
-I prompt (demande une fois avant de supprimer plus de trois fichiers)
            
1. $ rm notes.txt
# Supprime le fichier spécifié dans le dossier courant.
2. $ rm -rf dossier_inutile/
# Supprime récursivement un dossier et tout ce qu'il contient sans demander de confirmation.
3. $ rm -i *.jpg
# Demande confirmation pour chaque fichier jpg avant de le supprimer.
4. $ rm -v f1.txt f2.txt
# Supprime les deux fichiers et affiche une confirmation textuelle pour chacun.
5. $ rm file1.txt file2.txt
# Suppression simple de plusieurs fichiers ciblés.
6. $ rm -v log_*.txt
# Affiche "removed 'log_01.txt'", "removed 'log_02.txt'", etc. Utile pour confirmer ce qui a été effacé par un wildcard (*).
7. $ rm -rf folder/{sub1,sub2}/temp_*
# Utilisation des accolades (braces) pour cibler des fichiers spécifiques dans plusieurs sous-dossiers en une seule fois.
8. $ rm -- -file_starting_with_dash.txt
# Astuce : Le '--' indique à Linux que les arguments suivants ne sont pas des options, permettant de supprimer un fichier dont le nom commence par '-'.
9. $ find . -type f -name "*.tmp" -exec rm -f {} +
# Combine 'find' et 'rm' pour supprimer tous les fichiers .tmp dans l'arborescence actuelle de manière performante.
Astuce : Pour éviter les accidents, beaucoup d'administrateurs ajoutent alias rm='rm -i' dans leur fichier .bashrc. Ainsi, le système demande toujours confirmation par défaut.
Références : [rm @ GNU][rm @ man7]
cpCopy (Copier des données).
Duplique des fichiers ou des dossiers vers une nouvelle destination. `cp` est très utilisé pour les sauvegardes rapides, les duplications de configuration et les copies prudentes quand on veut éviter d'écraser l'existant.
-r recursive (obligatoire pour copier des répertoires et leur contenu)
-i interactive (demande avant d'écraser un fichier existant)
-p preserve (conserve les permissions, l'id propriétaire et les dates)
-v verbose (affiche le détail des copies effectuées)
-u update (copie seulement si la source est plus récente que la cible)
-n no-clobber (n'écrase jamais un fichier existant)
-a archive (équivalent à -dpR, idéal pour les sauvegardes)
-t target-directory (précise explicitement le dossier de destination)
--parents recrée le chemin source dans la destination
            
1. $ cp rapport.pdf rapport_sauvegarde.pdf
# Crée une copie exacte du fichier sous un nouveau nom dans le même dossier.
2. $ cp -r Documents/ /media/cle_usb/
# Copie tout le dossier Documents et ses sous-dossiers vers la clé USB.
3. $ cp *.jpg ~/Images/Photos/
# Copie toutes les images JPG du dossier courant vers le dossier Photos.
4. $ cp -u projet.c backups/
# Ne copie le fichier que s'il a été modifié depuis la dernière sauvegarde.
5. $ cp -a /etc/skel/ nouveau_home/
# Conserve droits, dates et liens. Le bon réflexe pour dupliquer une arborescence sans perdre ses métadonnées.
6. $ cp -n *.conf sauvegarde/
# Copie plusieurs fichiers de configuration sans écraser ceux déjà présents.
7. $ cp --parents ./conf/nginx/site.conf /tmp/test/
# Recrée le chemin source dans la destination, pratique pour préparer un dossier de test partiel.
Références : [cp @ GNU][cp @ man7][cp @ Malekal]
chmodChange Mode (Gestion avancée des permissions)
chmod définit les droits de lecture (r=4), écriture (w=2) et exécution (x=1) pour le propriétaire, le groupe et les autres. C'est la commande à connaître pour sécuriser un fichier sensible, rendre un script exécutable ou remettre des permissions cohérentes sur un dossier. 
-R Applique récursivement aux sous-dossiers et fichiers
-v Affiche chaque changement effectué
-c Affiche seulement les changements réellement appliqués
-f Supprime les messages d'erreur non critiques
--reference Copie les permissions d'un fichier modèle

u  utilisateur propriétaire
g  groupe propriétaire
o  autres utilisateurs
a  tout le monde

r  lecture
w écriture
x exécution
X exécution seulement si dossier ou fichier déjà exécutable
s bit setuid/setgid
t sticky bit, utile sur des dossiers partagés comme /tmp
		

                

                    Valeur
                    Droit
                    Symbole
                

                
7rwxLecture + Écriture + Exécution

                
6rw-Lecture + Écriture

                
5r-xLecture + Exécution

                
4r--Lecture seule

            
            
1. $ chmod +x script.sh
# Rend le fichier exécutable pour tout le monde (Propriétaire, Groupe, Autres).
2. $ chmod 755 dossier/
# rwxr-xr-x : Propriétaire a tous les droits, les autres peuvent seulement lire et entrer.
3. $ chmod -R 644 /var/www/html
# rw-r--r-- : Définit récursivement les droits de lecture standard pour des fichiers web.
4. $ chmod u=rwx,g=rx,o= mon_fichier
# Définit précisément : tout pour l'utilisateur, lecture/exécution pour le groupe, rien pour les autres.
5. $ chmod 644 index.html
# Standard pour les fichiers web : Propriétaire (6: rw-), Groupe (4: r--), Autres (4: r--).
6. $ chmod 755 scripts/
# Standard pour les répertoires : Tout le monde peut lister et entrer, seul le propriétaire modifie.
7. $ chmod 600 ~/.ssh/id_rsa
# Sécurisation de clé privée : Seul le propriétaire peut lire et écrire. Le système refuse souvent d'utiliser une clé trop exposée.
8. $ chmod u+x deploy.sh
# Ajoute (+) le droit d'exécution (x) uniquement à l'utilisateur propriétaire (u).
9. $ chmod g-w document.txt
# Retire (-) le droit d'écriture (w) au groupe (g).
10. $ chmod a+r readme.md
# Donne le droit de lecture (r) à tout le monde (a : all). Equivalent à ugo+r.
11. $ chmod -R u=rwX,go=r static/
# Usage avancé : Fixe (=) les droits récursivement. Le 'X' majuscule n'ajoute l'exécution qu'aux dossiers (pour pouvoir les parcourir) et non aux fichiers simples.
Références : [chmod @ linux.die][chmod @ IT-Connect][chmod @ GNU][chmod @ Malekal]
chownChange Owner (Gérer l'appartenance des fichiers)
chown permet de modifier l'utilisateur propriétaire et/ou le groupe propriétaire. Cette commande nécessite généralement les privilèges sudo et revient souvent lorsqu'on déploie un site, corrige un partage ou restaure une sauvegarde. 
-R recursive (applique le changement à tout le contenu d'un dossier)
-v verbose (affiche un message pour chaque fichier traité)
-c changes (signale uniquement les fichiers dont le propriétaire change)
-h no-dereference (modifie le lien symbolique lui-même, pas sa cible)
--reference file (copie le propriétaire d'un fichier existant)
            
1. $ sudo chown alice rapport.txt
# Change le propriétaire du fichier pour 'alice'. Le groupe reste inchangé.
2. $ sudo chown alice:developpeurs projet/
# Change à la fois le propriétaire (alice) et le groupe (developpeurs) du dossier.
3. $ sudo chown -R www-data:www-data /var/www/html
# Donne récursivement la propriété de tout le site web à l'utilisateur et au groupe 'www-data'.
4. $ sudo chown :admin script.sh
# Change uniquement le groupe pour 'admin' (notez le ':' avant le nom).
5. $ sudo chown --reference=file1.txt file2.txt
# Copie exactement les mêmes propriétaires de file1 vers file2. Évite les erreurs de saisie manuelle.
6. $ sudo chown -h alice lien_symbolique
# '-h' : Modifie le propriétaire du lien symbolique lui-même, et non du fichier vers lequel il pointe (comportement par défaut).
NB : Seul l'utilisateur root (ou via sudo) peut céder la propriété d'un fichier à un autre utilisateur pour des raisons de sécurité et de quotas disque.
Références : [chown @ linux.die][chown @ IT-Connect][chown @ GNU][chown @ Malekal]
findRecherche ultra-puissante dans l'arborescence.
find [chemin] [critères] [action]. C'est un outil de filtrage sur les métadonnées du système de fichiers (inode). Il est très utile pour retrouver des fichiers par nom, taille, date, propriétaire ou permissions, puis enchaîner avec une action.
-name pattern (par nom, sensible à la casse)
-iname pattern (par nom, insensible à la casse)
-type f/d (fichier / dossier)
-size n (par taille : +10M pour plus de 10 Mo, -1k pour moins de 1 ko)
-mtime n (par date de modification en jours : -7 pour les 7 derniers jours)
-user name (fichiers appartenant à un utilisateur précis)
-perm mode (par permissions exactes, ex: 777)
-exec command {} \; (exécute une commande sur chaque résultat trouvé)
-delete supprime directement les fichiers trouvés (à utiliser avec prudence !)
-maxdepth limite la profondeur de recherche
-empty cible les fichiers ou dossiers vides
            
1. $ find . -name "*.conf"
# Cherche tous les fichiers finissant par .conf dans le répertoire actuel et ses sous-dossiers.
2. $ find /var/log -type f -mtime -3
# Trouve les fichiers classiques modifiés au cours des 3 derniers jours dans /var/log.
3. $ find /tmp -user alice -exec rm {} \;
# Recherche tous les fichiers appartenant à alice dans /tmp et les supprime un par un.
4. $ find . -type f -size +100M
# Cherche uniquement des fichiers (-type f) de plus de 100 Mo dans le dossier actuel.
5. $ find /home -user alice
# Trouve tous les fichiers appartenant à l'utilisateur 'alice'.
6. $ find . -mtime -2
# Trouve les fichiers modifiés (-mtime) durant les dernières 48 heures.
7. $ find /tmp -type f -atime +10 -delete
# Trouve les fichiers n'ayant pas été accédés depuis plus de 10 jours et les supprime automatiquement.
8. $ find /etc -type f -name "*.conf"
# Trouve uniquement les fichiers (-type f) se terminant par .conf.
9. $ find /var/log -type d
# Liste uniquement les répertoires (-type d) dans le dossier des logs.
10. $ find /tmp -type f -empty
# Trouve tous les fichiers vides.
11. $ find . -perm 777
# Sécurité : Trouve les fichiers ayant des droits rwx pour tout le monde.
12. $ find /home -user alice -group developpeurs
# Recherche croisée par utilisateur ET par groupe.
13. $ find /var/log -name "*.log" -exec chmod 644 {} \;
# Cherche les logs et change leurs droits.
Le '{}' représente le fichier trouvé, '\;' termine la commande.
14. $ find /tmp -name "*.tmp" -delete
 # Action directe : Supprime tous les fichiers trouvés sans passer par -exec (plus rapide).
15. $ find . -maxdepth 1 -type f
# Limite la recherche au dossier courant sans descendre dans les sous-répertoires.
16. $ find /srv -type d -empty
# Repère les dossiers vides, souvent utiles à nettoyer ou à vérifier.
Conseil : Testez toujours votre commande avec -print ou sans action avant d'utiliser -delete ou -exec rm pour éviter de supprimer des données importantes par erreur.
Références : [Maîtriser Find @ stephane-robert][find @ it-connect (fr)][find man @ linux-die (en)][find @ Malekal]
chattrChange Attributes (Attributs système avancés).
Modifie des attributs bas niveau au sein du système de fichiers (souvent ext4).
+i immutable (rend le fichier impossible à modifier, supprimer, renommer ou lier)
+a append only (permet d'ajouter des données à la fin, mais pas de supprimer ou modifier l'existant)
+A no atime updates (ne met pas à jour la date de dernier accès, utile pour les performances)
+c compressed (demande au noyau de compresser le fichier sur le disque)
+s secure deletion (efface physiquement les données du disque lors de la suppression)
-R recursive (applique les attributs aux répertoires et à leur contenu)
-v version (affiche la version du programme)
            
1. $ sudo chattr +i /etc/passwd
# Rend le fichier des utilisateurs immuable. Même root ne pourra pas le modifier sans retirer l'attribut.
2. $ sudo chattr -i /etc/passwd
# Retire la protection d'immuabilité pour permettre de nouveau les modifications.
3. $ sudo chattr +a /var/log/syslog
# Autorise uniquement l'ajout de texte à la fin du fichier (idéal pour sécuriser les logs).
4. $ lsattr mon_fichier
# Utilise la commande complémentaire 'lsattr' pour visualiser les attributs actifs.

File & Directory Management - Lecture & Traitement

catConcatenate (Affichage, création et fusion de fichiers)
cat lit les données d'un ou plusieurs fichiers et les écrit sur la sortie standard (le terminal). Il est souvent couplé aux opérateurs de redirection > et >>. 
-n --number (numérote toutes les lignes)
-b --number-nonblank (numérote seulement les lignes remplies)
-s --squeeze-blank (condense les lignes vides multiples en une seule)
-v --show-nonprinting (affiche les caractères de contrôle non imprimables)
-e equivalent to -vE
-E  --show-ends : display $ at end of each line
-A --show-all (affiche tout : fins de ligne, tabulations et caractères invisibles)
            
1. $ cat /etc/hostname
# Affiche simplement le nom de la machine à l'écran.
2. $ cat -n script.sh
# '-n' (number) : Affiche le contenu avec les numéros de lignes. Indispensable pour le débugging.
3. $ cat fichier1.txt >> fichier_existant.txt
# Ajoute (append) le contenu du premier fichier à la fin du second sans l'écraser.
4. $ cat > nouveau_fichier.txt
# Crée un fichier et attend que vous tapiez du texte. Appuyez sur Ctrl+D pour enregistrer et quitter.
5. $ cat fichier1.txt fichier2.txt > fusion.txt
# CONCATÉNATION : Fusionne le contenu des deux fichiers dans un troisième (écrase le contenu de fusion.txt).
6. $ cat -s logs.txt
# Nettoie l'affichage d'un fichier de log en supprimant les blocs de lignes vides inutiles.
7. $ cat /dev/null > mon_fichier.log
# Vide instantanément le contenu d'un fichier (pratique pour réinitialiser un log volumineux).
8. $ cat -A fichier_windows.txt
# Permet de voir les caractères cachés '^M' (CRLF) qui causent souvent des erreurs de script.
9. $ cat -e config.conf
# '-e' : Affiche un '$' à la fin de chaque ligne. Permet de repérer les espaces traînants invisibles.
10. $ cat <<EOF > print.sh
	#!/bin/bash
	echo \$PWD
	echo $PWD
	EOF
   # Le fichier print.sh contiendra:
	#!/bin/bash
	echo $PWD
	echo /home/user
# Permet d'écrire plusieurs lignes dans un fichier directement depuis un script ou le terminal jusqu'à rencontrer la balise 'EOF'.
12. $ cat /var/log/syslog | grep "error"
# Envoie le contenu du log vers grep pour filtrer les erreurs.
Note de performance : Pour lire de très gros fichiers, préférez less ou tail. cat tente de tout charger/afficher d'un coup, ce qui peut saturer votre terminal.
Références : [cat @ Malekal][cat @ linux.die][cat @ GNU][cat @ man7]
tacReverse Cat (Affichage de fichiers à l'envers)
tac concatène et affiche les fichiers en inversant l'ordre des lignes. La dernière ligne du fichier devient la première affichée dans le terminal. 
-b Ajoute un séparateur avant chaque ligne
-r Interprète le séparateur comme une expression régulière
-s Choisit le séparateur de blocs ou de lignes
			
1. $ tac /var/log/syslog
# Affiche le journal système en commençant par les entrées les plus récentes (utile si le fichier est court).

2. $ tac fichier.txt > inverse.txt
# Crée un nouveau fichier où l'ordre des lignes est strictement inversé par rapport à l'original.

3. $ tac /var/log/auth.log | grep "Failed password" | head -n 5
# Recherche les 5 dernières tentatives de connexion échouées en remontant le fichier de log.

4. $ tac -s "SECTION" notes.txt
# '-s' (separator) : Utilise un mot ou un caractère spécifique comme délimiteur au lieu du saut de ligne (\n). 
   # Inverse l'ordre des blocs séparés par le mot "SECTION".

5. $ ls -1 | tac
# Liste les fichiers du répertoire courant (un par ligne) et inverse l'ordre alphabétique manuellement.
Différence avec tail -f : Alors que tail -f attend les nouvelles lignes en continu, tac traite le fichier existant à l'instant T pour en inverser la lecture totale.
Références : [tac @ GNU][tac @ man7]
head / tailLire le début ou la fin d'un fichier.
Indispensable pour les fichiers énormes (logs de plusieurs Go) que l'on ne veut pas charger entièrement en mémoire. `head` donne un aperçu rapide, `tail` est parfait pour voir les dernières lignes et surveiller un fichier actif.
-n Définit le nombre de lignes à afficher
-c Travaille en nombre d'octets au lieu des lignes
-f Avec `tail`, suit le fichier en temps réel
-F Suit le fichier même après rotation de log
-q / -v Masque ou force l'affichage du nom du fichier
			
1. $ head -n 20 /etc/passwd
# Affiche uniquement les 20 premières lignes du fichier.
2. $ tail -n 50 /var/log/apache2/error.log
# Affiche les 50 dernières erreurs enregistrées par le serveur web.
3. $ tail -f /var/log/auth.log
# L'option '-f' (follow) est vitale : elle laisse le fichier ouvert et affiche les nouvelles lignes EN DIRECT au fur et à mesure qu'elles s'écrivent. Utile pour surveiller une attaque ou un crash.
4. $ tail -F /var/log/nginx/access.log
# Plus robuste que `-f` si le fichier de log est renommé ou recréé par logrotate.
5. $ head -c 64 fichier.bin
# Affiche les 64 premiers octets. Pratique pour jeter un oeil à un fichier binaire ou à un en-tête.
Références : [tail @ Malekal][head @ GNU][tail @ man7][head @ man7]
less / morePaginateurs (Lecture interactive).
Permettent de lire un fichier page par page. `less` est supérieur car il permet de remonter en arrière, de chercher un motif, d'afficher les numéros de ligne et même de suivre un log.
/motif Recherche en avant dans le fichier
?motif Recherche en arrière
-N Affiche les numéros de ligne
-S Désactive le retour à la ligne automatique
+F Mode follow, proche de `tail -f`
			
1. $ less /var/log/syslog
# Ouvre le fichier en mode lecture. Utilisez 'Espace' pour descendre, 'b' pour remonter, '/' pour chercher un mot, et 'q' pour quitter.
2. $ command | less
# Pipe (redirige) la sortie d'une commande longue vers less pour pouvoir l'analyser tranquillement.
3. $ less -N /etc/ssh/sshd_config
# Affiche les numéros de ligne, utile lorsqu'on doit ensuite modifier le fichier avec un éditeur.
4. $ less +F /var/log/syslog
# Suit le fichier comme un log vivant, puis `Ctrl+C` permet de revenir au mode lecture normal.

grepRecherche de motifs textuels (Le couteau suisse).
'grep' (Global Regular Expression Print) filtre les lignes correspondant à une recherche. C'est l'un des outils les plus puissants de Linux pour lire des logs, fouiller une arborescence de configuration ou retrouver une chaîne dans du code.
-i Ignore la casse
-r Recherche récursive dans un dossier
-v Inverse la sélection
-n Affiche les numéros de ligne
-E Active les expressions régulières étendues
-c Compte le nombre de lignes correspondantes
-w Cherche un mot entier uniquement
-A / -B / -C Affiche le contexte avant/après la ligne trouvée
			
1. $ grep "error" /var/log/syslog
# Affiche toutes les lignes contenant le mot exact "error".
2. $ grep -i "failed" /var/log/auth.log
# '-i' (ignore-case) cherche sans différencier minuscules et majuscules.
3. $ grep -r "192.168.1.1" /etc/nginx/
# '-r' (récursif) cherche l'IP dans tous les fichiers de configuration du dossier nginx.
4. $ grep -v "^#" config.conf
# '-v' (invert-match) affiche tout SAUF ce qui correspond. Ici, on cache les commentaires (lignes commençant par #).
5. $ grep -E "error|critical" log.txt
# '-E' (Extended Regex) permet d'utiliser des opérateurs logiques comme 'OU' (|).
6. $ grep -Rn "server_name" /etc/nginx/
# Recherche récursive avec numéros de ligne dans toute une arborescence de configuration.
7. $ grep -C 2 "ERROR" app.log
# Affiche 2 lignes avant et après chaque correspondance pour garder le contexte.
Références : [grep @ Malekal][grep @ GNU][grep @ man7]
sedStream Editor (Édition automatique de masse).
Permet de modifier du texte "au vol" sans ouvrir le fichier manuellement. Idéal pour l'automatisation, les remplacements en masse et le nettoyage rapide de fichiers de configuration.
-i Modifie directement le fichier
-n N'affiche pas automatiquement chaque ligne
-e Ajoute une commande sed supplémentaire
-E Active les expressions régulières étendues
-f Lit les commandes depuis un script sed
			
1. $ sed 's/localhost/127.0.0.1/g' file.txt
# 's' pour substitute. Remplace toutes ('g' pour global) les occurrences de 'localhost' par l'IP. (Affiche le résultat à l'écran).
2. $ sed -i 's/PORT=80/PORT=8080/' config.env
# '-i' (in-place) modifie DIRECTEMENT le fichier source. Très puissant, soyez sûr de votre commande !
3. $ sed '5,10d' file.txt
# 'd' pour delete. Supprime les lignes 5 à 10 du flux de sortie.
4. $ sed -n '1,20p' fichier.txt

# Utilise une expression régulière pour garder la clé et remplacer seulement la valeur.
Références : [sed @ GNU][sed @ man7][sed @ Malekal]
awkTraitement de texte par colonnes.
`awk` voit un fichier comme un tableau. Chaque mot est une variable ($1, $2, etc.). C'est parfait pour extraire des colonnes, filtrer des lignes et construire de petits rapports très vite.
-F Définit le séparateur de champs
-f Lit le programme awk depuis un fichier
-v var=val Passe une variable au script awk
BEGIN / END Blocs exécutés avant ou après le traitement
$1, $2... Référence les colonnes/champs
			
1. $ awk '{print $1, $3}' data.txt
# Affiche uniquement la première et la troisième colonne de chaque ligne.
2. $ awk -F':' '{print $1}' /etc/passwd

4. $ df -h | awk 'NR>1 {print $1, $5}'
# Produit une vue très compacte du filesystem et de son taux d'occupation.
5. $ awk 'BEGIN {FS=\":\"} {print NR, $1, $7}' /etc/passwd
# Construit un mini rapport avec numéro de ligne, login et shell par défaut.

psPhotographier les processus à un instant T.
`ps` n'est pas interactif : il capture l'état des processus au moment exact où vous lancez la commande. C'est parfait pour filtrer, trier et intégrer le résultat dans un script ou un diagnostic rapide.

2. Gestion du Réseau (Focus Connectivité)

ipConsulter les interfaces, adresses IP et routes.
`ip` remplace les anciens `ifconfig`, `route` et `arp`. C'est la commande moderne de référence pour lire ou ajuster rapidement l'état réseau d'une machine Linux.
addr show Affiche les adresses IP des interfaces
link show Affiche l'état des interfaces (UP/DOWN, MTU...)
route show Affiche la table de routage
-4 / -6 Force IPv4 ou IPv6
-c Active les couleurs
link set dev IFACE up|down Active ou coupe une interface
			
1. $ ip -c addr show
# Affiche clairement toutes les interfaces et leurs IP.
2. $ ip route show
# Permet d'identifier la passerelle par défaut (`default via ...`).
3. $ ip -4 addr show dev eth0
# Filtre sur une interface et uniquement en IPv4.
4. $ sudo ip link set dev eth0 down
# Désactive l'interface. Faire l'inverse avec `up` pour la réactiver.
Références : [ip @ LinuxTricks][diagnostic réseau @ IT-Connect][ip @ man7]
ssLister les sockets et les ports ouverts.
`ss` est le remplaçant moderne de `netstat`. Il permet de voir rapidement quels ports sont en écoute, quelles connexions sont établies, et quel processus tient chaque socket.
-l Affiche uniquement les sockets en écoute
-t Filtre sur TCP
-u Filtre sur UDP
-n Garde les IP/ports en numérique
-p Montre le processus lié à la socket
-a Affiche écoute + connexions établies
			
1. $ ss -tulpn
# La commande réflexe pour savoir quel service écoute sur quel port.
2. $ ss -ant
# Affiche les connexions TCP actives, sans résolution de noms.
3. $ ss -ltnp
# N'affiche que les sockets TCP en écoute avec le processus associé.
4. $ ss -uap
# Très utile pour inspecter les sockets UDP ou UNIX.
Références : [ss @ LinuxTricks][netstat/ss @ IT-Connect][ss @ man7]
pingTester la joignabilité et la latence réseau.
`ping` est le premier test de base quand un hôte semble inaccessible. Il vérifie la connectivité IP et donne une idée de la latence et des pertes de paquets. Attention : certains équipements bloquent l'ICMP.
-c Limite le nombre de paquets envoyés
-i Définit l'intervalle entre deux envois
-4 / -6 Force IPv4 ou IPv6
-s Change la taille du payload
-W Fixe le timeout d'attente d'une réponse
			
1. $ ping -c 4 1.1.1.1
# Vérifie rapidement si la sortie Internet fonctionne sans dépendre du DNS.
2. $ ping -c 4 google.com
# Teste à la fois la résolution DNS et la connectivité.
3. $ ping -4 -c 4 example.com
# Force le test en IPv4 si l'IPv6 perturbe le diagnostic.
4. $ ping -i 0.2 -c 20 serveur.local
# Permet d'observer la stabilité et une éventuelle perte de paquets sur une courte période.
Références : [ping @ IT-Connect][diagnostic réseau @ IT-Connect][ping @ man7]
digInterroger le DNS et vérifier les résolutions.
`dig` sert à savoir ce qu'un serveur DNS répond réellement. Il est indispensable quand un nom de domaine ne se résout pas, répond vers la mauvaise IP, ou présente un problème de propagation.
+short Affiche uniquement la réponse utile
@serveurDNS Interroge un serveur DNS précis
-t TYPE Demande un type d'enregistrement précis (A, AAAA, MX...)
-x Fait une résolution inverse à partir d'une IP
+trace Suit toute la chaîne de résolution DNS
			
1. $ dig +short example.com
# Retourne uniquement l'adresse IP trouvée, sans le bruit du reste.
2. $ dig @1.1.1.1 example.com
# Compare la réponse d'un résolveur spécifique à celle du DNS local.
3. $ dig -t MX example.com
# Affiche les enregistrements mail du domaine.
4. $ dig -x 8.8.8.8
# Résolution inverse : utile en diagnostic réseau ou mail.
Références : [diagnostic réseau @ IT-Connect][dig @ man7][outils réseau modernes @ Red Hat]
ncTester un port TCP/UDP et manipuler un flux réseau brut.
`nc` (netcat) est une vraie boîte à outils réseau. En administration, on l'utilise très souvent pour tester si un port est ouvert, écouter un flux ou transférer rapidement un fichier entre deux machines.
-z Mode scan : teste sans envoyer de données applicatives
-v Mode verbeux
-u Travaille en UDP au lieu de TCP
-l Passe en écoute côté serveur
-w Définit un timeout
-p Précise le port d'écoute selon les implémentations
			
1. $ nc -zv 192.168.1.10 22
# Vérifie si le port SSH est joignable.
2. $ nc -zvu 192.168.1.10 53
# Test rapide d'un port UDP, ici DNS.
3. $ nc -l -p 1234
# Met la machine en écoute sur le port 1234 pour recevoir un flux.
4. $ nc -w 2 192.168.1.10 1234 < fichier.txt
# Envoie le contenu d'un fichier vers la machine en écoute.
Références : [nc @ LinuxTricks][nc @ man7]

dfVoir l'espace disponible par système de fichiers.
`df` répond à la question "quelle partition est pleine ?". Il affiche l'occupation de chaque système de fichiers monté, ce qui en fait le réflexe numéro un quand un serveur signale un manque d'espace disque.
-h Affichage lisible en K/M/G
-T Affiche le type de système de fichiers
-i Affiche l'utilisation des inodes
-x TYPE Exclut un type de filesystem (tmpfs, squashfs...)
-a Inclut aussi les filesystems à 0 bloc
			
1. $ df -h
# Vue standard de l'espace libre et utilisé sur tous les points de montage.
2. $ df -hT
# Ajoute le type de filesystem, utile pour distinguer ext4, xfs, tmpfs, etc.
3. $ df -i
# Détecte un problème d'inodes épuisés, fréquent avec énormément de petits fichiers.
4. $ df -h -x tmpfs -x squashfs
# Nettoie l'affichage pour se concentrer sur les vrais volumes de stockage.
Références : [df @ LinuxTricks][df @ man7]
duMesurer la taille des dossiers et trouver les plus lourds.
`du` complète `df` : une fois qu'on sait quel volume est saturé, `du` permet de découvrir quels répertoires ou fichiers prennent réellement la place.
-s Affiche uniquement le total du chemin ciblé
-h Affichage lisible en K/M/G
-a Inclut aussi les fichiers, pas seulement les dossiers
--max-depth=N Limite la profondeur d'analyse
-x Reste sur le filesystem courant
			
1. $ du -sh /var/log
# Taille totale du dossier de logs.
2. $ du -h --max-depth=1 /home
# Affiche la taille de chaque dossier utilisateur au premier niveau.
3. $ du -ah /var | sort -hr | head -n 20
# Liste les fichiers/dossiers les plus lourds dans /var.
4. $ du -xhd 1 /
# Analyse la racine sans traverser d'autres systèmes de fichiers montés.
Références : [du @ LinuxTricks][du @ man7]
lsofLister les fichiers et sockets ouverts par les processus.
`lsof` est redoutable pour comprendre quel processus tient un fichier, un port ou un montage occupé. C'est souvent la commande qui débloque un `umount: target is busy` ou qui identifie le programme derrière un port réseau.
-i Filtre sur les sockets réseau
-i :PORT Filtre sur un port précis
-u Filtre par utilisateur
-p Filtre par PID
+c 15 Allonge la largeur du nom de commande
+D Liste les fichiers ouverts sous un répertoire
			
1. $ lsof -i :80
# Identifie le processus qui écoute ou utilise le port 80.
2. $ lsof +D /var/log/nginx
# Montre quels processus gardent ouverts des fichiers dans ce dossier.
3. $ lsof -u mysql
# Isole tout ce que l'utilisateur mysql a actuellement ouvert.
4. $ lsof /dev/sdb1
# Très utile avant un démontage de volume bloqué.
Références : [lsof @ LinuxTricks][lsof @ man7][lsof exemples]
uptimeConnaître le temps de fonctionnement et la charge moyenne.
`uptime` donne en une ligne trois informations cruciales : depuis combien de temps la machine tourne, combien d'utilisateurs sont connectés, et surtout les `load average` sur 1, 5 et 15 minutes.
-p Affichage lisible du temps de fonctionnement
-s Affiche l'heure exacte de démarrage
-V Version de l'outil
			
1. $ uptime
# Affiche la ligne complète avec load average 1/5/15 min.
2. $ uptime -p
# Retour plus lisible, par exemple "up 3 days, 4 hours".
3. $ uptime -s
# Donne la date/heure exacte du dernier démarrage.
Références : [uptime @ LinuxTricks][uptime @ Tecmint][uptime @ man7]
unameAfficher des informations sur le noyau et l'architecture.
`uname` sert surtout à identifier le noyau Linux et l'architecture matérielle. C'est très utile pour vérifier la compatibilité d'un paquet, d'un binaire, ou documenter précisément l'environnement système.
-a Affiche toutes les informations principales
-r Version du noyau
-m Architecture matérielle (x86_64, aarch64...)
-n Nom réseau de la machine
-o Nom du système d'exploitation
			
1. $ uname -a
# Vue complète du noyau, version, hostname et architecture.
2. $ uname -r
# Affiche uniquement la release du noyau Linux.
3. $ uname -m
# Très utile pour savoir si la machine est en x86_64, arm64, etc.
4. $ uname -o
# Affiche le nom de l'OS, souvent `GNU/Linux`.
Références : [uname @ LinuxTricks][uname @ Linuxize][uname @ man7]

4. Statistiques avancées

vmstatObserver mémoire, CPU, swap et I/O sur la durée.
`vmstat` est excellent pour détecter les symptômes d'une saturation progressive : swap qui grimpe, CPU en attente d'I/O, files d'exécution qui grossissent, etc. Il est particulièrement utile lorsqu'un serveur ralentit sans cause évidente.
-a Affiche mémoire active/inactive
-s Affiche un résumé détaillé des compteurs
-w Élargit l'affichage pour éviter les colonnes tronquées
-t Ajoute un timestamp
DELAY COUNT Rafraîchit périodiquement un nombre donné de fois
			
1. $ vmstat 1 5
# Cinq relevés espacés d'une seconde pour observer une tendance récente.
2. $ vmstat -w 2 10
# Même principe avec colonnes élargies, plus lisible sur grosses machines.
3. $ vmstat -s
# Résumé global utile pour un rapport rapide.
4. $ vmstat -t 1 5
# Ajoute l'horodatage à chaque ligne, pratique pour corréler avec d'autres logs.
Références : [vmstat @ LinuxTricks][vmstat @ PhoenixNAP][vmstat @ man7]
mpstatLire l'utilisation CPU globale et cœur par cœur.
`mpstat` fait partie de `sysstat`. Il est précieux sur les machines multi-cœurs, car il montre si la charge est équilibrée ou si un seul CPU est saturé alors que les autres restent calmes.
-P ALL Affiche les stats pour chaque CPU
-u Affiche l'utilisation CPU
-I SUM Affiche les interruptions agrégées
DELAY COUNT Rafraîchit périodiquement
			
1. $ mpstat
# Donne la moyenne CPU globale depuis le démarrage ou depuis le dernier échantillon.
2. $ mpstat -P ALL 1 5
# Montre l'activité de chaque cœur pendant 5 secondes.
3. $ mpstat -u 2 3
# Se concentre sur l'utilisation CPU avec trois relevés espacés de 2 secondes.
4. $ mpstat -I SUM 1 3
# Utile pour voir si une tempête d'interruptions dégrade la machine.
Références : [mpstat @ Tecmint][mpstat @ man7]
iostatAnalyser l'activité disque et l'attente I/O.
`iostat` est indispensable quand un système semble lent à cause du stockage. Il aide à voir si les disques sont sollicités, si le temps d'attente (`await`) grimpe, ou si une partition spécifique concentre l'activité.
-d Affiche les statistiques de périphériques
-x Affichage étendu (await, util, svctm...)
-m Affiche les débits en Mo/s
-p DEVICE Inclut les partitions d'un périphérique
-y Ignore le premier rapport depuis le boot
DELAY COUNT Rafraîchit périodiquement
			
1. $ iostat
# Vue d'ensemble CPU + I/O depuis le démarrage.
2. $ iostat -x 1 5
# Le mode le plus utile pour diagnostiquer un stockage lent en temps quasi réel.
3. $ iostat -mx 2 5
# Affiche les débits en Mo/s avec détails étendus.
4. $ iostat -y -p sda 1 3
# Ignore le premier relevé historique et cible un disque précis avec ses partitions.
Références : [iostat @ Tecmint][iostat exemples][iostat @ man7]

1. Services (Systemd), logs et paquets

systemctlGérer les services système (démons).
`systemctl` est la commande-clé de systemd. Elle sert à démarrer, arrêter, activer au boot et diagnostiquer les services comme SSH, nginx, Docker ou MariaDB.
start Démarre un service immédiatement
stop Arrête le service proprement
restart Redémarre le service
status Affiche l'état, le PID, les derniers logs
enable Active le lancement automatique au démarrage
disable Désactive le lancement automatique
daemon-reload Recharge les unit files après modification
list-units --type=service Liste les services chargés
			
1. $ sudo systemctl start nginx
# Démarre nginx tout de suite, sans modifier son comportement au prochain boot.
2. $ sudo systemctl enable --now ssh
# Active le service au démarrage ET le lance immédiatement.
3. $ systemctl status docker
# Le meilleur premier réflexe pour voir si un service tourne et pourquoi il échoue.
4. $ sudo systemctl daemon-reload
# A lancer après modification d'un fichier `.service` pour que systemd relise sa configuration.
5. $ systemctl list-units --type=service --state=failed
# Affiche rapidement les services en erreur sur la machine.
Références : [systemctl @ systemd][systemctl @ man7][systemd @ LinuxTricks]
journalctlConsulter les logs centralisés du système.
Le journal systemd agrège les messages du noyau, des services et du système. `journalctl` permet de filtrer par service, priorité ou plage temporelle.
-u Filtre par unit/service
-f Suit les nouveaux logs en temps réel
-p Filtre par niveau (err, warning, info...)
--since / --until Filtre par date ou période
-b Affiche uniquement le boot courant
-xe Montre le contexte d'erreur le plus utile
-n Limite au nombre de lignes voulu
--disk-usage Affiche l'espace disque utilisé par le journal
			
1. $ sudo journalctl -u nginx -n 50
# Affiche les 50 dernières lignes de logs du service nginx.
2. $ sudo journalctl -f
# Equivalent à un `tail -f` sur les logs système.
3. $ sudo journalctl -p err -b
# Filtre les erreurs du boot en cours. Très utile juste après un redémarrage problématique.
4. $ sudo journalctl --since "2026-04-09 08:00" --until "2026-04-09 10:00"
# Permet de cibler une fenêtre horaire précise pendant une investigation.
Références : [journalctl @ systemd][journalctl @ man7][journalctl @ LinuxTricks]
apt / dnfInstaller, mettre à jour et rechercher des paquets.
La gestion des paquets fait partie des bases de l'administration Linux. `apt` est courant sur Debian/Ubuntu, `dnf` sur Fedora/RHEL modernes. Les réflexes sont similaires : rechercher, installer, mettre à jour, supprimer.
apt update Actualise l'index des paquets disponibles
apt install Installe un paquet
apt upgrade Met à jour les paquets installés
apt remove / purge Supprime un paquet (purge retire aussi sa config)
apt search Recherche dans les dépôts
dnf install Installe un paquet sous Fedora/RHEL
dnf upgrade Met à jour le système
dnf info Affiche les détails d'un paquet
			
1. $ sudo apt update && sudo apt upgrade
# Séquence classique sur Ubuntu/Debian : on rafraîchit l'index puis on applique les mises à jour.
2. $ sudo apt install curl
# Installe l'outil `curl` avec ses dépendances.
3. $ apt search openssh
# Recherche les paquets dont le nom ou la description contient `openssh`.
4. $ sudo dnf install htop
# Equivalent Fedora/RHEL pour installer `htop`.
5. $ sudo apt purge apache2
# Supprime le paquet et sa configuration locale. A utiliser seulement si on veut repartir proprement.
Références : [apt @ Ubuntu][dnf @ docs][apt @ Malekal]

2. Archivage & compression

tarCréer, lister et extraire des archives.
`tar` regroupe plusieurs fichiers dans une seule archive `.tar`. On lui ajoute souvent une compression comme gzip (`.tar.gz`) ou xz (`.tar.xz`) pour gagner de la place.
-c Create : crée une archive
-x Extract : extrait une archive
-t List : liste le contenu sans extraction
-f Précise le nom du fichier archive
-z Active la compression gzip
-J Active la compression xz
-C Extrait ou archive depuis un dossier cible
--exclude Ignore certains chemins
			
1. $ tar -cvf archive.tar dossier/
# Archive simple, sans compression. Rapide mais peu économe en espace.
2. $ tar -czvf backup.tar.gz /etc /var/www
# Format très courant pour une sauvegarde de configuration et de site web.
3. $ tar -xvf archive.tar -C /tmp/restore
# Extrait l'archive dans un dossier précis plutôt que dans le répertoire courant.
4. $ tar -tf backup.tar.gz
# Vérifie le contenu d'une archive avant extraction.
5. $ tar -czvf logs.tar.gz /var/log --exclude='*.gz'
# Archive les logs non déjà compressés pour éviter les doublons.
Références : [tar @ GNU][tar @ man7][tar @ LinuxTricks][tar/zip/gzip @ Malekal]
zip / gzipCompression rapide et formats courants.
`zip` est très pratique pour échanger des fichiers avec Windows. `gzip` compresse un fichier unique, souvent des logs ou des exports, et s'intègre très bien avec `tar`.
zip -r Compresse récursivement un dossier
zip -e Ajoute un mot de passe à l'archive ZIP
unzip Extrait une archive ZIP
gzip Compresse un fichier en `.gz`
gunzip Décompresse un fichier `.gz`
zcat Affiche le contenu d'un `.gz` sans le décompresser
gzip -k Conserve le fichier source en plus du `.gz`
gzip -d Equivalent à `gunzip`
			
1. $ zip -r archive.zip projet/
# Crée une archive ZIP complète d'un dossier avec son arborescence.
2. $ unzip archive.zip -d /tmp/projet
# Extrait dans un dossier de destination précis.
3. $ gzip access.log
# Remplace `access.log` par `access.log.gz` pour économiser de l'espace.
4. $ zcat access.log.gz | grep 500
# Recherche les erreurs HTTP 500 dans un log compressé sans le restaurer sur disque.
Références : [gzip @ GNU][zip @ man7][compression @ Malekal]

3. Disques, tâches planifiées et utilitaires d'admin

mount / umountMonter et démonter des systèmes de fichiers.
Un disque, une partition ou une image ISO devient accessible en l'attachant à un point de montage. Le réflexe sécurité : toujours `umount` avant de débrancher un support.
mount Affiche les montages actifs
-t Spécifie le type de système de fichiers
-o Passe des options (ro, rw, loop, noexec...)
umount Démonte un système de fichiers
lsblk Liste les disques et points de montage
blkid Affiche UUID et type de filesystem
mount -a Monte tout ce qui est déclaré dans `/etc/fstab`
			
1. $ lsblk
# Identifie les disques, partitions et points de montage existants avant toute action.
2. $ sudo mount /dev/sdb1 /mnt/usb
# Monte la partition `sdb1` dans le dossier `/mnt/usb`.
3. $ sudo mount -o loop image.iso /mnt/iso
# Monte un fichier ISO comme un vrai disque pour consulter son contenu.
4. $ sudo umount /mnt/usb
# Démonte proprement le support. Si ça refuse, vérifier qu'aucun shell ou processus ne l'utilise.
Références : [mount @ man7][umount @ man7][mount/umount @ Malekal]
ddCopie bit à bit et conversion bas niveau.
`dd` copie les données de manière brute. Extrêmement puissant, mais dangereux : inverser `if` et `of` peut détruire un disque entier. Toujours relire la commande avant validation.
if= Input file : source
of= Output file : destination
bs= Taille des blocs copiés
count= Nombre de blocs à copier
status=progress Affiche l'avancement
conv=fsync Force l'écriture physique avant fin
			
1. $ sudo dd if=ubuntu.iso of=/dev/sdc bs=4M status=progress conv=fsync
# Crée une clé USB bootable à partir d'une image ISO.
2. $ sudo dd if=/dev/sda of=/backup/disque.img bs=64K status=progress
# Clone brut d'un disque entier vers un fichier image.
3. $ sudo dd if=/dev/zero of=test.bin bs=1M count=100
# Génère un fichier de test de 100 Mo, utile pour des essais de stockage.
Références : [dd @ GNU][dd @ man7][dd @ Malekal]
crontabPlanifier des tâches automatiques.
`cron` exécute des commandes à heure fixe ou à intervalle régulier. Bon réflexe : toujours utiliser des chemins absolus dans les scripts cron, car l'environnement est minimal.
-e Edite la crontab de l'utilisateur courant
-l Liste les tâches planifiées
-r Supprime la crontab courante
-u Cible un autre utilisateur (avec sudo/root)
* * * * * Format minute heure jour mois jour-semaine
			
1. $ crontab -e
# Ouvre l'éditeur pour ajouter ou modifier vos tâches planifiées.
2. $ crontab -l
# Permet de vérifier rapidement ce qui est déjà programmé.
3. 0 4 * * * /home/user/backup.sh
# Lance le script tous les jours à 04:00.
4. */5 * * * * /usr/bin/curl -fsS http://localhost/health
# Exécute un contrôle toutes les 5 minutes.
Références : [crontab @ man7][format crontab @ man7][cron/crontab @ LinuxTricks]
hostnamectl / timedatectlGérer l'identité de la machine, l'heure et le fuseau.
Ces deux commandes systemd sont très pratiques en administration courante. `hostnamectl` gère le nom de la machine, tandis que `timedatectl` permet de vérifier ou corriger l'heure, la timezone et l'activation de la synchronisation NTP.
hostnamectl Affiche le nom d'hôte, l'OS, le noyau et l'architecture
hostnamectl set-hostname Définit un nouveau nom de machine
timedatectl Montre l'heure locale, UTC, timezone et l'état NTP
timedatectl list-timezones Liste les fuseaux disponibles
timedatectl set-timezone Change le fuseau horaire
timedatectl set-ntp true Active la synchro automatique de l'heure
			
1. $ hostnamectl
# Donne rapidement l'identité du système et quelques infos utiles de diagnostic.
2. $ sudo hostnamectl set-hostname srv-web-01
# Renomme proprement la machine avec un nom plus explicite.
3. $ timedatectl
# Vérifie si l'heure et le fuseau sont corrects, ce qui est crucial pour les logs et certificats.
4. $ sudo timedatectl set-timezone Europe/Paris
# Change le fuseau horaire du serveur.
5. $ sudo timedatectl set-ntp true
# Active la synchronisation automatique pour éviter la dérive de l'horloge système.
Références : [hostnamectl @ systemd][timedatectl @ systemd][hostnamectl @ man7][timedatectl @ man7][timedatectl @ IT-Connect]
nohup / &Lancer un processus persistant en arrière-plan.
Quand un terminal se ferme, ses processus enfants reçoivent souvent un signal de fin. `nohup` permet de les laisser tourner. En complément, `&`, `jobs`, `bg` et `fg` aident à gérer les processus du shell.
nohup Ignore le signal HUP envoyé à la fermeture du terminal
& Lance la commande en arrière-plan
jobs Liste les tâches du shell courant
bg Reprend une tâche suspendue en arrière-plan
fg Ramène une tâche au premier plan
			
1. $ nohup ./script.sh &
# Lance le script en fond et le laisse tourner même après la fermeture de la session.
2. $ nohup python3 -m http.server 8080 > server.log 2>&1 &
# Exemple classique : on redirige la sortie dans un fichier plutôt que `nohup.out`.
3. $ jobs
# Affiche les commandes déjà lancées depuis le shell courant.
Références : [nohup @ GNU][nohup @ man7][nohup @ Malekal]
mkfsFormater une partition avec un système de fichiers.
`mkfs` construit un système de fichiers neuf sur une partition. C'est une action destructive : toutes les données présentes sur la cible seront perdues. Bien vérifier le nom du périphérique avec `lsblk` avant de lancer la commande.
mkfs.ext4 Crée un filesystem ext4
mkfs.xfs Crée un filesystem XFS
mkfs.vfat Crée un filesystem FAT32
-L Définit un label lisible
-n Label (selon l'outil, ex. vfat)
blkid Vérifie ensuite le type et l'UUID obtenus
			
1. $ lsblk
# Toujours identifier la bonne partition avant un formatage.
2. $ sudo mkfs.ext4 -L DATA /dev/sdb1
# Formate `/dev/sdb1` en ext4 et lui attribue le label `DATA`.
3. $ sudo mkfs.vfat -n USB32 /dev/sdc1
# Formate une clé USB en FAT32, pratique pour l'échange avec plusieurs OS.
4. $ sudo blkid /dev/sdb1
# Contrôle final : type de filesystem, UUID et label.
Références : [mkfs @ man7][mkfs.ext4 @ man7][mkfs @ Malekal]

Network Administration

sshConnexion distante sécurisée à une machine Linux.
`ssh` est la base de l'administration à distance. Il chiffre la session, permet l'authentification par mot de passe ou clé publique, et peut aussi rediriger des ports.
-p Port SSH à utiliser
-i Clé privée à employer
-L Redirection locale de port
-J Passe par un bastion / jump host
-v Mode verbeux pour le debug
			
1. $ ssh user@192.168.1.20
# Ouvre une session distante sur la machine cible.
2. $ ssh -i ~/.ssh/id_ed25519 admin@serveur
# Force l'utilisation d'une clé privée spécifique.
3. $ ssh -L 8080:127.0.0.1:80 user@serveur
# Le port local 8080 est relié au port 80 du serveur distant via un tunnel chiffré.
4. $ ssh -v user@serveur
# Mode debug pratique si la connexion échoue (clé, DNS, refus, algo, etc.).
Références : [ssh @ man7][OpenSSH manuals][ssh @ Malekal]
scp / rsyncCopier ou synchroniser des fichiers à distance.
`scp` est simple et rapide pour une copie ponctuelle. `rsync` est meilleur pour les synchronisations répétées, les gros volumes et les reprises partielles.
scp -r Copie récursive d'un dossier
scp -P Port SSH personnalisé
rsync -a Mode archive (permissions, dates, liens...)
rsync -v Affiche le détail des transferts
rsync -z Compresse pendant le transfert
rsync --delete Supprime la destination absente à la source
			
1. $ scp fichier.txt user@serveur:/tmp/
# Copie un fichier local vers le serveur distant.
2. $ scp -r ./site user@serveur:/var/www/
# Copie un dossier complet via SSH.
3. $ rsync -avz ./backup/ user@serveur:/srv/backup/
# Synchronisation efficace en préservant les droits et les dates.
4. $ rsync -av --delete ./public/ user@serveur:/var/www/html/
# Rend la destination identique à la source. A manipuler avec prudence à cause de `--delete`.
Références : [scp @ man7][rsync manual][scp @ IT-Connect][rsync @ IT-Connect]
curl / wgetTester une URL, télécharger un fichier ou interroger une API.
`curl` est idéal pour les tests HTTP, les API REST et les en-têtes de réponse. `wget` est pratique pour les téléchargements simples ou récursifs.
curl -I Récupère uniquement les en-têtes HTTP
curl -O Enregistre sous le nom distant
curl -L Suit les redirections
curl -X Spécifie la méthode HTTP
curl -H Ajoute un en-tête
wget -O Choisit le nom du fichier de sortie
wget -c Reprend un téléchargement interrompu
			
1. $ curl -I https://example.com
# Vérifie le code HTTP, le type MIME, le serveur, la longueur, etc.
2. $ curl -L -O https://example.com/archive.tar.gz
# Télécharge un fichier en suivant les redirections.
3. $ curl -X POST -H "Content-Type: application/json" -d '{"ok":true}' https://api.example.com/test
# Exemple d'appel API REST avec payload JSON.
4. $ wget -c https://releases.example.com/big.iso
# Reprend le téléchargement si la connexion a été interrompue.
Références : [curl manual][wget @ GNU][curl @ Malekal][wget @ Malekal]
traceroute / tracepath / mtrAnalyser le chemin réseau et repérer où la connectivité se dégrade.
Quand `ping` montre une latence anormale ou qu'un accès distant ne répond pas, ces outils aident à comprendre où le trafic bloque. `traceroute` et `tracepath` suivent les sauts, `mtr` combine ping et traceroute dans un mode interactif très parlant.
traceroute Affiche les routeurs traversés jusqu'à la cible
traceroute -n Désactive la résolution DNS pour aller plus vite
tracepath Alternative souvent présente sans privilèges root
mtr Mesure en continu perte et latence par saut
mtr -rw Produit un rapport texte lisible pour diagnostic
			
1. $ traceroute example.com
# Montre les différents sauts entre votre machine et la destination.
2. $ traceroute -n 8.8.8.8
# Plus rapide pour un test brut car on évite la résolution des noms d'hôtes intermédiaires.
3. $ tracepath 1.1.1.1
# Alternative simple pour observer le chemin sans options complexes.
4. $ mtr -rw google.com
# Produit un mini rapport avec perte de paquets et latence par hop, très utile en support réseau.
Références : [traceroute @ man7][tracepath @ man7][mtr @ mankier][traceroute @ IT-Connect]
host / nslookupVérifier rapidement la résolution DNS d'un nom ou d'un enregistrement.
Pour les problèmes de connectivité, il faut souvent distinguer une panne réseau d'un simple souci DNS. `host` est compact et lisible, `nslookup` reste encore très courant dans les tutos et environnements mixtes.
host Résout un nom de domaine simplement
host -t MX Interroge un type d'enregistrement précis
host -t NS Liste les serveurs DNS autoritaires
nslookup Outil de requête DNS interactif ou direct
nslookup domain server Interroge un serveur DNS donné
			
1. $ host example.com
# Donne l'adresse IP associée au domaine.
2. $ host -t MX gmail.com
# Affiche les serveurs de messagerie d'un domaine.
3. $ host -t NS example.com
# Montre quels serveurs sont autoritaires pour la zone DNS.
4. $ nslookup example.com 1.1.1.1
# Permet de comparer la réponse d'un résolveur public avec celle du réseau local.
Références : [host @ man7][nslookup @ man7][host @ Malekal][nslookup @ Malekal]
nmapScanner des ports et inventorier les services exposés.
`nmap` est un outil de diagnostic et d'inventaire réseau très puissant. Il permet de voir quels ports sont ouverts, quels services répondent et parfois leur version. A utiliser uniquement sur les machines et réseaux autorisés.
-p Scanne un ou plusieurs ports précis
-sV Tente d'identifier les versions de services
-O Essaye de deviner l'OS
-Pn Ignore l'étape de ping initiale
-A Scan agressif : scripts, versions, OS, traceroute
			
1. $ nmap 192.168.1.10
# Scan de base des ports les plus courants.
2. $ nmap -p 22,80,443 192.168.1.10
# Vérifie rapidement quelques ports d'intérêt.
3. $ sudo nmap -sV 192.168.1.10
# Essaie d'identifier les services et leurs versions.
4. $ sudo nmap -A 192.168.1.10
# Analyse plus poussée, utile pour un audit interne ou un lab.
Références : [nmap guide][nmap @ IT-Connect]
